# Handoff — 2026-04-10

## Session Summary

Major rewrite of the terminal/agent system. Replaced raw PTY transport with tmux-managed sessions, replaced sentinel pattern with tmux wait-for, added settings system, garbage collector, and comprehensive edge case handling.

## What Was Built

### Terminal Bridge Rewrite (from 2026-04-09 handoff)
- `ManagedBridge` struct — single bridge per task, abort-based cancellation
- Bridge tracking in `SessionRegistry` — set/cancel/has_active_bridge
- Shell ready detection (prompt regex + ANSI stripping) — later replaced by tmux

### tmux Integration (the big change)
- `TmuxTransport` (`tmux_transport.rs`) — new ChatTransport impl wrapping tmux sessions
- Each task gets `bentoya_{task_id}` tmux session, `tmux attach` spawned in PTY for xterm.js
- `tmux resize-window` for SIGWINCH propagation to TUI apps
- `tmux send-keys -l` for command injection (literal mode + separate Enter)
- `tmux wait-for` for completion detection (replaces sentinel echo pattern)
- Session persistence across app restarts (tmux sessions survive crashes)
- Recovery on startup — discovers existing sessions, re-registers or kills orphans

### Settings System
- `~/.bentoya/settings.json` with defaults (max_sessions, gc intervals, default CLI/model)
- Cached in memory via `OnceLock<Mutex>` — no disk I/O per access
- Workspace config column overrides global settings
- API endpoints: `GET/POST /api/settings`
- Resolution chain: trigger config > task.model > workspace config > global settings

### Pipeline Fixes
- CLI-aware prompt flag (`-p` for claude, positional for codex)
- `agent_session.status` updated on completion (exit criteria checks this)
- `create_pr` trigger pushes branch before `gh pr create`
- `.task.md` written to worktree (token optimization pointer)
- Default prompt references `.task.md` instead of full spec

### MCP Server Hardening
- `require_app()` gate on all 8 mutation handlers
- Health check validates response body (not just HTTP 200)
- No silent DB fallback in production — clear error messages
- Test-only fallback via `allow_db_fallback()`

### Agent Cancellation
- Moving task out of trigger column to non-trigger column sends Ctrl+C
- Shared `cancel_task_agent()` function in tmux_transport
- Skips cancel if target also has trigger (new agent replaces old)
- Column guard in `spawn_cli_trigger_task` — mark_complete skipped if task moved

### Garbage Collector (`gc.rs`)
- Periodic (every 5 min configurable): kills orphaned tmux sessions, idle sessions > 4h
- Detects running agents with dead tmux sessions — marks failed
- Lenient date parsing (RFC3339, space-separated, naive)

### E2E Tests Fixed
- 17/17 webdriver tests passing
- DOM text assertions → IPC assertions (kanban CSS hides text from webdriver)
- Trigger isolation: save/clear/restore around pipeline tests
